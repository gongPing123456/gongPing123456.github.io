Sql 插入速度问题

2021-07-10 Sql

🤓简单写一下，后续会深入Mybatis 和Mysql

在使用Mybatis-plus savaBatch 插入 1000条左右数据，会相应很久的时候

开始以为是@Transactional的问题，😄的解决，结果发现是单纯的时间太长

接下来 ，重新开一个简单的系统测试一下。🍳

准备

创建一个用于插入的数据库。

创建一个User 的插入结构，

使用For循环插入1000条数据

@Test
	void contextLoads() {
		long current=System.currentTimeMillis();

		for (int i = 0; i < 1000; i++) {
			User user=new User();
			user.setName(System.currentTimeMillis()+"");
			user.setNum(i);

		}
		long end=System.currentTimeMillis();
		log.info("插入时间："+(end-current));

	}

插入时间：78611

使用Mybatis-plus saveBatch 插入 1000条数据

public void plusTest(){

			long current=System.currentTimeMillis();

        this.executeBatch((sqlSession) -> {

插入时间：922

时间对比

For 插入：78611

Mybatis-plus 插入: 36904

ForEach 插入：922

结论：插入1000 条 数据，将数据拼接，通过sql 批量插入速度更快。

😇 插入少量数据-10条

For 插入：1809

Mybatis-plus 插入: 1289

ForEach 插入：838

结论：少量插入数据，时间差距不大。
